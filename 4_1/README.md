## Обязательная задача 1

Есть скрипт:
```bash
a=1
b=2
c=a+b
d=$a+$b
e=$(($a+$b))
```

Какие значения переменным c,d,e будут присвоены? Почему?

| Переменная  | Значение | Обоснование |
| ------------- | ------------- | ------------- |
| `a`  |   1  | неявное объявление переменной с присвоением значения |
| `b`  |   2  | неявное объявление переменной с присвоением значения |
| `c`  | a+b  | bash неявное объявление переменной 'с' с присвоением строкового значения 'a+b'|
| `d`  | 1+2  | аналогично работае со значениями переменных производя конкатенацию значений в строку|
| `e`  |  3   | в данном случае (()) - говорит bash что выполняется арифметическая операция над значениями переменных, bash приводит их к int и складывает |

## Обязательная задача 2
На нашем локальном сервере упал сервис и мы написали скрипт, который постоянно проверяет его доступность, записывая дату проверок до тех пор, пока сервис не станет доступным (после чего скрипт должен завершиться). В скрипте допущена ошибка, из-за которой выполнение не может завершиться, при этом место на Жёстком Диске постоянно уменьшается. Что необходимо сделать, чтобы его исправить:
```bash
while ((1==1)
do
    curl https://localhost:4757
    if (($? != 0))
    then
	date >> curl.log
    fi
done
```

```bash
  if (($? != 0))
  then

#выход из цикла при доступности сервиса
  else
    echo "Сервис доступен с [$(date)]|" >> curl.log
    break
  fi

```

Необходимо написать скрипт, который проверяет доступность трёх IP: `192.168.0.1`, `173.194.222.113`, `87.250.250.242` по `80` порту и записывает результат в файл `log`. Проверять доступность необходимо пять раз для каждого узла.

### Ваш скрипт:

```bash
iva@c8:~/test $ cat test5.sh
#!/usr/bin/env bash

declare -a hosts=(192.168.0.1 173.194.222.113 87.250.250.242)
port=80
count=5

for ip in ${hosts[@]}
do
  for i in $(seq $count)
  do
    curl "http://${ip}:${port}" 1>/dev/null 2>&1
    if (($? != 0))
    then
      echo "[$(date)]|${ip}:${port}|не доступен" >> curl.log
    else
      echo "[$(date)]|${ip}:${port}|доступен" >> curl.log
    fi
  done
done
iva@c8:~/test $ 
```
